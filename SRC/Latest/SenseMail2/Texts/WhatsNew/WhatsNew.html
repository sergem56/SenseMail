    Ver.2.00

    There are lots of changes in this version.
    We added a background check for the new messages. But that is not a push message from the mail server. It is a system's background task that is fired from time to time, depending on the app usage. The more often you open the app, the more frequent is the check performed. The check period might be, for example, 10 minutes, might be half an hour, or might be even once a day. We cannot keep the active connection to your mail server to get the alert right away due to the operating system's limitation. At the same time, we do not want to keep that connection active from our servers as, in that case, we need to send your data to servers, other than your mail server, that is against our policy.

  • Dark mode
  • Customizable background color for every account
  • Added a "Print" action
  • Added some more pre-defined search filters
  • Added an emergency clear-all PIN code
  • Added a "Clear all" button to the settings
  • Added memory overwriting and cleaning on exit
  • Fixed a slow-motion attachment
  • Optimized network usage
  • Numerous stability improvement
  • Other bugfixes and improvements

    Ver.1.9

  • Added Biometric ID. You can login to one account with TouchID or FaceID.
  • Added message auto-load. When you scroll the list to the end, the next set of messages are loaded automatically.
  • Now you can encrypt a message on the server. If you received a plain-text message, you can leave it on the server in the encrypted format. Please, read our blog for more details on how to do it securely.
  • Added an option to send a message if an SSL certificate of a server is invalid.
  • Added a setting to sort the list with newly loaded messages. It can be sorted by date as a whole list or just a part being added. In other words, when you load the next part of messages from multiple email accounts, they can be mixed (sorted by date) with already shown messages or added to the end of that list to make it easier to spot them. This applies to the list of several email accounts. If there is only one account, all the messages will be sorted by date anyway.
  • Changed a message view layout for better readability - subject is on top in bold
  • Added from, to, reply-to and size info, expanding by a tap on a sender's address in message view
  • Detect html for encrypted messages to show them as html (before encrypted messages were treated as plain text and if you encrypt an existing message, its formatting messed up)
  • HTML-Text switch to view a message
  • Other minor bug fixes and improvements

    Ver.1.8

  • To enhance security we introduced a mutable key. We add a random number of PBKDF2-SHA512 rounds to the key derivation function in a range between 250K and 700K iterations. For every message sent you'll have a different key derived from your password or a fixed certificate.
  • An option to set the number of messages to load from each account. Default is ten
  • Added “On read” expiration time for OTC. Now you can send a message that will vanish after first read. Such messages will have a red lock icon in the list
  • Fixed OTC exchange bug appearing if you send the same certificates twice
  • Other minor bug fixes and improvements
    •

    Ver.1.7. What's new?

  • One-Time Certificates. Exchange the certificates with your counterpart and get the highest level of protection for your messages. One key is used for one message. Each key used can be set to vanish at a specified day.
  • Added What's new page. You are reading it right now. You can call it any time later from the main menu.
  • Added high/normal priority for a message. A small blue/red dot to the right of the sender's address.
  • Added search important for messages with high priority
  • Added read receipt handling. A receipt's icon beside the subject line.
  • Changed icons of the messages in the list. Lock - encrypted with a password. Double lock - encrypted with a one-time certificate. "Burning" double lock - the cert is going to expire. "Cracked" double lock - the certificate has expired and the message is unreadable. Green sender - the message is encrypted with a certificate.
  • The source code was checked against the OWASP (The Mobile Application Security Verification Standard) requirements
  • Rename folder on long press on a folder in the list
  • No exceptions from the system's networking security rules - no more insecure loads over http
  • Various bug fixes and improvements

    Ver.1.6

    There are quite a lot of changes in this version. Lets start chronologically and list major fixes.

  • We've added an easy setup feature. Now to add an account you need to enter your e-mail address and we'll try to figure out the server settings. For Gmail you'll be redirected to the web-authorisation form, for other accounts you'll need to enter a password.

  • Now you can enable an auto-signature for an e-mail account.

  • There was a bug – imagine that you compose a message and get a call. After finishing a call you notice that the letter has disappeared. Annoying. Now your letter will be hidden to an icon and won't be lost. The same was with a secure note. Fixed.
  • In the list of messages you can tap on the table header to sort the table by date, by unread and by the sender in a cycle
  • Added message search. Since we cannot search message body as it is encrypted, we added search for unread, starred, large messages, protected and answered messages. Text search will look into sender and subject (for unprotected messages) fields. Also, when the date is detected, you can search messages before/on/after that date
  • Added HTML or plain text view for a secure note
  • Now you can save documents to the secure gallery
  • There is an option for large font for list of the messages
  • We've added a multiple selection in message list. Now you can select multiple messages and delete, copy, move or flag them all at once
  • We fixed a number of other bugs
